# /focotx — Enfocar mente en lo técnico

Cambia la inclinación del agente hacia pensamiento **técnico** (backend Python, engine, analizadores, base de datos, DeepSeek, rendimiento, caché, deuda técnica).

No solo lee el roadmap — el agente debe:
1. Revisar el proyecto con ojos de ingeniero: ¿hay code smells? ¿módulos acoplados? ¿cache bien usado? ¿prompt de DeepSeek eficiente?
2. Identificar mejoras técnicas invisibles para UX/UI
3. Leer `documentos/roadmap.md` para prioridades formales
4. Considerar: optimizaciones DeepSeek (temperature, max_tokens, prompt), estructura de módulos, migraciones DB, deuda técnica, tests faltantes
5. Reportar: próximos pasos técnicos + mejoras detectadas + items pendientes del roadmap

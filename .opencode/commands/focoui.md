# /focoui — Enfocar mente en el diseño visual

Cambia la inclinación del agente hacia pensamiento **de diseño** (dashboard UI, layouts, paletas, tipografía, componentes visuales, animaciones, responsive, gráficos Plotly).

No solo lee guías de UI — el agente debe:
1. Revisar el proyecto con ojos de diseñador: ¿la interfaz del dashboard es coherente? ¿los templates siguen un sistema de diseño? ¿hay problemas de accesibilidad?
2. Identificar mejoras visuales: paletas, espaciado, estados hover/focus, animaciones, modales, responsive mobile
3. Leer `templates/` y `static/` para entender el estado actual
4. Considerar: tema oscuro (colores existentes), gráficos Plotly, Bootstrap u otros frameworks, adaptación mobile
5. Reportar: próximos pasos de UI + mejoras detectadas
